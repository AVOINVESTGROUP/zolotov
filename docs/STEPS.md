# Журнал разработки Zolotov

## Шаг 1: Фундамент проекта (2026-04-05)

### Реализовано:
- **Дизайн-система**: Переход на Tailwind v4 `@theme` в `globals.css`. Настройка люкс-палитры и шрифтов (Cormorant Garamond, Inter, Montserrat).
- **Конфигурация Next.js**: Добавлена поддержка домена изображений `zolotov.store` в `next.config.ts`.
- **Типизация**: Создан `types/index.ts` с интерфейсами WooCommerce (Product, Category, Order) и CartItem.
- **Утилиты**: Реализованы `cn` для классов и `formatPrice` для валюты AED в `lib/utils.ts`.
- **API Клиент**: Настроен `lib/woo.ts` на базе Axios для безопасных серверных запросов к WooCommerce API v3.
- **Состояние**: Реализован стор корзины на Zustand с персистентностью в localStorage (`store/cart.ts`).
- **Структура**: Созданы директории для компонентов (`layout`, `ui`, `sections`, `product`).

## Шаг 2: Вёрстка главной страницы "Тёмный Люкс" (2026-04-05)

### Реализовано:
- **Header**: Адаптивный компонент с логотипом "ZOLOTOV", прозрачностью при скролле (Framer Motion) и меню из `zolotov.store`.
- **Hero**: Полноэкранная секция с заголовком "Platinum 950 & Diamonds" и акцентом на Дубай.
- **ProductCard**: Минималистичная карточка товара с центрированным текстом и тёмным фоном для изображений (как в JewelryMuse).
- **Сетка товаров**: 4-колонная композиция на главной странице (`app/page.tsx`).
- **Блок доверия**: Секция с преимуществами (бесплатная доставка, гарантия GIA).

### Следующие шаги:
- Создание модального окна карточки товара (Quick View).
- Вёрстка страницы каталога с фильтрами.
- Реализация страницы корзины.

## Шаг 3: Telegram Mini App & Автоматизация (2026-04-05)

### Реализовано:
- **TWA SDK**: Интеграция `@twa-dev/sdk`. Настройка `TWAProvider` для управления MainButton, BackButton и Haptic Feedback.
- **Двуязычность**: Легкая система `MiniAppI18nProvider` (RU/EN) с автоопределением языка пользователя Telegram.
- **Интерфейс TMA**: Созданы страницы Главная, Каталог, Детали товара, Корзина, Checkout и Успех внутри `/miniapp`.
- **Навигация**: Полная изоляция лейаута мини-приложения от основного сайта.

## Шаг 4: API Proxy & Безопасность (2026-04-06)

### Реализовано:
- **Proxy Endpoints**: Созданы серверные маршруты `/api/products`, `/api/categories`, `/api/products/[slug]`.
- **Безопасность**: Скрытие `WC_CONSUMER_KEY` и `SECRET` от клиентской части через проксирование запросов.
- **Рефакторинг**: Перевод всех компонентов Mini App на использование внутреннего API вместо прямого обращения к WooCommerce.

### Следующие шаги:
- Деплой на Vercel и финальное тестирование в Telegram.
- Настройка приветственного сообщения (Start Message) в BotFather.
- Интеграция Telegram Payments (опционально по запросу).
